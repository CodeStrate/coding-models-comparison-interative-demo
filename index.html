Concurrency Bug: Cache Race Condition

A threaded cache with fetch_fn had a TOCTOU race: two threads could both miss the cache and make duplicate fetches. Models were asked to identify and fix the bug.

BUGGY CODE 
cache = {}

def get_cached(key, fetch_fn):
    if key not in cache:
        cache[key] = fetch_fn(key)  # Race condition!
    return cache[key]

# Two threads calling get_cached("user_1", fetch_from_db)
# simultaneously may both fetch — defeating the cache purpose.

Why Gemma's Fix Loses Its Purpose

By locking the entire function body, Gemma's version makes every thread wait — including during the potentially slow fetch_fn call. Double-checked locking (Devstral Small 2/Codestral) avoids this by making cache hits lock-free.

The Clever Explanation Gemma Gave

Gemma's Thread A/B walkthrough was genuinely the clearest of all three models. The explanation quality was top-tier. The fix just didn't match the explanation's precision.

The Real Answer

No model is perfect. But Devstral Small 2 is the only one in this set that delivers correct reasoning + working code consistently. Gemma punches well above its weight at ~7.5B parameters, but its concurrency fix demonstrates that confidence can mask incorrectness. Codestral is a specialist — great at autocomplete, unreliable at reasoning.

Devstral Small 2 — Primary

Best for: Agentic coding, refactoring, async logic

Use Devstral Small 2 when you need code that works the first time. Its reasoning precision on concurrency and async correctness makes it the safest choice for production systems.

Gemma E4B — Watchlist

Best for: FIM, edge-case detection, low-resource deployment

Remarkable efficiency. Ideal where you need good-but-not-perfect code with minimal parameters. Always review Gemma's output — its verbosity can mask logic errors.

Codestral — Specialist

Best for: IDE autocomplete, FIM completion, pattern matching

Trust Codestral for fill-in-middle tasks and architectural scaffolding. Don't trust it for async correctness, error handling, or any task requiring runtime precision.
